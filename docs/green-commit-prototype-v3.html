오픈소스가 뭐예요? 🌍

코딩은 좀 하지만 “오픈소스 기여”가 뭔지 감이 안 온다면 — 3분이면 충분해요.

What · 오픈소스가 뭔가요

누구나 코드를 볼 수 있고, 누구나 개선에 참여할 수 있게 공개된 프로젝트예요. 네가 매일 쓰는 브라우저·라이브러리·앱의 상당수가 오픈소스로 만들어졌어요.

전 세계 개발자들이 조금씩 고쳐가며 함께 키우는 거예요. 그리고 그 “조금”에 너도 낄 수 있어요.

이런 것도 다 ‘기여’예요
✓문서의 오타 한 글자 고치기
✓영어 문서를 한국어로 번역
✓예제 코드 하나 추가
거창한 기능이 아니어도 돼요. 작은 것부터가 진짜 시작이에요.
How · 기여는 이렇게 굴러가요
1 Fork
내 계정으로 복사→ 2 고치기
내 복사본에서→ 3 PR
반영 요청→ 4 리뷰
메인테이너 피드백→ 5 Merge
원본에 합쳐짐 🎉
복잡해 보여도 괜찮아요. Green Commit이 단계마다 옆에서 코치하고, 익숙해진 단계는 자동화·건너뛰기도 돼요.

GitHub 계정으로 시작하기

가입은 GitHub 하나로 끝. 이때 받은 권한으로 나중에 Fork·PR을 자동화할 수 있어요.

이 권한을 받아요 (읽기 + 기여 동작)
✓공개 저장소·활동 읽기 (추천·분석용)
✓내 Fork 생성 / PR 열기 (자동화용)
×비공개 저장소·이메일은 건드리지 않아요

권한은 언제든 설정에서 해제할 수 있어요. 자동화는 항상 사용자가 버튼을 눌러 확인한 뒤에만 실행돼요.

로그인 / 회원가입

다른 로그인 수단은 제공하지 않아요. 기여 작업이 전부 GitHub에서 일어나기 때문에 GitHub 계정 하나로 통일했어요.

이제 커밋하고 PR로 반영 요청해요

학습한 대로 IDE에서 직접 고치고, 아래 안내대로 커밋 → PR 하면 첫 기여 완료예요.

1. 커밋 (변경 기록)
git checkout -b fix/short-description git commit -am "Fix ..." git push origin HEAD
좋은 메시지: Fix typo in instructor help page · 피할 것: update, fix, asdf
2. Pull Request (반영 요청)
What: (무엇을 바꿨는지) Why: (왜 필요한지) How tested: (어떻게 확인했는지) Fixes #(이슈 번호)

‘Fixes #번호’를 넣으면 머지 시 이슈가 자동으로 닫혀요.
